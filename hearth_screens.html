1. Приветствие
Первый экран. Эмоциональный, не информационный.
Что на экране
  • Иллюстрация / логотип Hearth
  • Одна строка — суть продукта
  • Кнопка «Создать аккаунт»
  • Ссылка «Уже есть аккаунт»
Чего нет
  • Feature tour / карусель слайдов
  • Объяснений механики
  • Permission-запросов

2. Регистрация / Вход
Уже готово — не трогаем.
Статус
Экраны login и sign_up существуют. Оставляем как есть.
🪪
3. Имя и аватар
Как тебя будет видеть семья.
Что на экране
  • Поле: имя (как в приложении)
  • Выбор аватара из набора
  • Возможность загрузить фото
Примечание
  • Прототипы edit_profile_avatar уже есть
  • Без выбора роли — роль не нужна в механике
🏠
4. Создать дом или вступить
Ключевой экран — определяет кто хозяин.
Создать дом
  • Название дома
  • Аватар / иллюстрация дома
  • Кнопка «Создать» → ты хозяин
Вступить
  • Поле ввода кода-приглашения
  • Или QR-скан
  • После — ты участник, не хозяин
Права хозяина: настройки дома приглашать участников удалять задачи любого
Доступны с любого экрана — топ-бар и FAB
➕
Создать задачу (FAB)
Центральная кнопка нижнего бара. Один экран — два режима.
🔴 Базовый минимум (сверху)
  • Название задачи
  • Исполнитель: я / другой / доска
  • ← если я → личная задача, цены нет
  • ← если другой → задача с наградой
  • Цена в манне (только для наград)
⚪ Расширенный максимум (снизу)
  • Срочность: Anytime / Scheduled / Urgent
  • Дата и время
  • Адрес / место
  • Travel time буфер
  • Описание / инструкция
Тип задачи: исполнитель = я → личная исполнитель = другой → с наградой
🔔
Уведомления
Колокольчик в топ-баре. Все события системы.
Типы уведомлений
  • Тебе назначена задача (Proposed)
  • Задача принята исполнителем
  • Задача выполнена — нужен Approve
  • Твой Approve → манна начислена
  • Предложена новая награда
  • Куплена твоя награда
Действия прямо из уведомления
  • Approve / Reject задачи
  • Принять задачу
  • Одобрить предложение награды
  • Перейти к объекту
👤
Профиль
Аватар в топ-баре. Личное + управление домом.
Для всех
  • Имя и аватар
  • Общий баланс манны
  • История задач
  • Настройки уведомлений
  • Управление семьёй (участники, приглашения)
Только для хозяина дома
  • Настройки дома (название, аватар)
  • Удаление участника
  • Удаление любой задачи
Таб 1 — Главная: твой командный центр на сегодня
🏠
Главная
Один экран для всех — содержимое адаптируется под то что актуально для тебя прямо сейчас.
Секция 1 — Срочное
  • Задачи со статусом Urgent
  • Задачи Completed, ждущие моего Approve
  • Просроченные задачи
Секция 2 — Сегодня
  • Мои задачи на сегодня (Scheduled)
  • Личные задачи из тасктрекера
  • Полоска семейного календаря
Секция 3 — Манна
  • Мой суммарный баланс манны
  • Чья манна и сколько (мини-карточки)
  • Тап → переход в Награды / магазин этого человека
Секция 4 — Семья
  • Участники и их активные задачи
  • Кто что сейчас делает
Переходы: Детали задачи Approve / Reject Магазин человека
Принцип: Главная — это «что требует внимания прямо сейчас», не полный обзор всего. Длинные списки живут в Задачах и Наградах.
Таб 2 — Задачи: два типа, три суб-таба
✅
Задачи — суб-таб «Мои»
Всё что назначено мне + мои личные задачи.
Личные задачи (создатель = я)
  • Простой чекбокс-список
  • Без манны, без Approve
  • Статусы: В работе / Готово
  • Возможно отдельная секция сверху
Задачи с наградой (я — исполнитель)
  • Статус: Proposed / Accepted / Overdue
  • Кто создал + цена манны
  • Принять / Отметить готовым
📋
Задачи — суб-таб «Доска»
Unassigned задачи — может взять любой.
Что отображается
  • Все задачи со статусом Unassigned
  • Кто создал + цена манны
  • Срочность
  • Кнопка «Взять»
После «Взять»
  • Задача переходит в Proposed
  • Появляется в «Мои»
  • Создатель получает уведомление
⏳
Задачи — суб-таб «Жду ответа»
Задачи которые я создал и которые ждут моего действия.
Что здесь
  • Completed — жду моего Approve
  • Proposed — исполнитель ещё не принял
  • Accepted — в работе у исполнителя
Действия
  • Approve → манна начислена
  • Reject → задача назад в Accepted + комментарий
  • Редактировать / удалить (если ещё не принята)
📄
Детали задачи
Push-экран. Разный в зависимости от типа задачи и роли.
Если я — исполнитель
  • Название, описание, инструкция
  • Кто создал + цена манны
  • Срок, место, travel time
  • Кнопки: Принять / Отметить готовым
Если я — создатель
  • Статус исполнения
  • Кто исполнитель
  • Кнопки: Approve / Reject (если Completed)
  • Редактировать / удалить
Таб 3 — Неделя: семейная логистика
📅
Неделя
Семейный календарь с travel time буфером. Одинаков для всех участников.
Основной вид
  • Горизонтальная полоса дней (7 дней)
  • Временная сетка по вертикали
  • Цвет события = цвет участника
  • Аватары участников на событии
Travel time
  • Буфер перед событием с адресом
  • Тап на адрес → навигатор
  • Настраивается при создании события
Переходы: Детали события Создать событие
Открытый вопрос: Задачи с дедлайном — они тоже отображаются в календаре как события, или только в Задачах?
Таб 4 — Награды: peer-to-peer магазин на основе персональной манны
🏆
Награды — главный экран
Два раздела: моя манна (чужие магазины) + мой магазин.
Секция «Моя манна»
  • Карточка каждого участника семьи
  • На карточке: аватар + баланс его манны у меня
  • Тап → Магазин этого человека
  • Нулевой баланс = нет доступных наград
Секция «Мой магазин»
  • Награды которые я выставил
  • Каждая: название, цена, кол-во покупок
  • Кнопка «Добавить награду»
  • Входящие предложения наград от других
🛍️
Магазин участника
Push-экран. Открывается тапом на карточку манны.
Что видно
  • Имя и аватар участника
  • Мой баланс его манны (сколько могу потратить)
  • Список его наград с ценами
  • Кнопка «Купить» — активна если хватает манны
После покупки
  • Моя манна этого человека уменьшается
  • Манна сгорает (не переходит к нему)
  • Владелец магазина получает уведомление
  • Выполняет обещание в реальной жизни
💡
Предложить награду
Модал. Просишь кого-то добавить нужную тебе награду в свой магазин.
Форма
  • Название желаемой награды
  • Описание (почему хочу)
  • Кому предлагаю (участник)
Что происходит
  • Уведомление получателю
  • Получатель одобряет + ставит цену
  • Награда появляется в его магазине
  • Предложивший видит статус «Ожидает»
Открытый вопрос: Кто может одобрять предложения наград — только хозяин дома, или любой участник чьему магазину предложили?
✏️
Создать / редактировать награду
Модал. Добавление награды в свой магазин.
Поля
  • Название
  • Описание
  • Цена в моей манне
  • Иконка / картинка
  • Лимит покупок (необязательно)
Ключевая идея Наград: Это не один общий магазин — это набор персональных магазинов. Ты тратишь манну Маши только в Машином магазине, манну Папы — только в Папином. Каждый сам решает что предложить в обмен на выполнение своих задач.
